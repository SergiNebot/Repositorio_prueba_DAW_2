Este es el orden de los comandos del github:
1. git init (inicializa el repositorio)
2. git remote add <nombre> <url> (agrega un repositorio remoto (en caso de no haberlo agregado previamente))
3. git branch add <nombre> (crea una nueva rama y se mueve a ella (en caso de ser necesario))
4. git add . (agrega todos los archivos modificados)
5. git commit -m "<nombre del commit>" (confirma los cambios)
6. git push origin <nombre_rama> (sube los cambios al repositorio remoto)
7. git status (muestra el estado del repositorio)
8. git log --oneline (muestra el historial de commits de forma resumida (en caso de querer ver los commits desde visualizaci√≥n)

Opcionales:
  - git remote -v (muestra los repositorios remotos)
  - git pull origin <nombre_rama> (descarga los cambios del repositorio remoto y los fusiona con la rama actual)
  - git checkout <nombre_rama> (cambia a otra rama)
  - git merge <nombre_rama> (fusiona la rama especificada con la rama actual)
  - git branch (muestra las ramas existentes y en cual se encuentra actualmente)
  - git branch -m <nombre_rama> (cambia el nombre de la rama principal a "main" o el nombre que se desee)
